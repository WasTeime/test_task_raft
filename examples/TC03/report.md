## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Синтаксис Swift: переменные, типы данных, операторы
- Управление потоком: условия, циклы, switch
- Функции и замыкания
- Структуры данных: массивы, словари, кортежи
- Объектно-ориентированное программирование: классы, структуры, перечисления

**Маршрут:**
- Начинаешь с синтаксиса Swift — без этого нельзя написать ни одной программы. Учишься объявлять переменные, работать с типами данных (Int, String, Bool), выполнять базовые операции. После этого шага ты пишешь простые скрипты которые обрабатывают числа и текст.
- Скрипты нужно управлять — нужны условия и циклы. Изучаешь if-else, switch, for, while. После этого шага ты пишешь программы которые принимают решения и повторяют действия.
- Код растёт — нужно его организовать. Разбиваешь логику на функции, учишься передавать параметры и возвращать результаты. Изучаешь замыкания (closures) для обработки данных. После этого шага есть переиспользуемые блоки кода которые легко вызывать.
- Данные нужно структурировать — работаешь с массивами, словарями, кортежами. Учишься фильтровать, сортировать, преобразовывать коллекции. После этого шага ты обрабатываешь наборы данных и готовишь их для отображения.
- Код становится сложнее — нужна архитектура. Изучаешь классы, структуры, перечисления, наследование, протоколы. Учишься моделировать реальные объекты (User, Post, Product). После этого шага есть типизированная модель данных которую можно использовать в приложении.

**Milestone:** Могу написать консольное приложение на Swift которое читает данные, обрабатывает их через функции и коллекции, выводит результат.


**Практика:**
- **Калькулятор расходов** — Консольное приложение которое принимает список расходов (категория, сумма), вычисляет итого по категориям, выводит отчёт. Используешь массивы, словари, функции для группировки данных.
- **Управление списком контактов** — Программа которая хранит контакты (имя, телефон, email), позволяет добавлять, удалять, искать контакты. Используешь структуры для моделирования контакта, массивы для хранения, функции для операций.
- **Анализатор текста** — Приложение которое читает текст, подсчитывает слова, буквы, находит самое частое слово, выводит статистику. Используешь строки, словари для подсчёта, функции для анализа.

### Фаза 2: Practice (30 дней)

**Темы:**
- UIKit: UIViewController, UIView, навигация
- Работа с экранами: создание интерфейса программно и через Storyboard
- Обработка пользовательского ввода: UIButton, UITextField, UITableView
- Жизненный цикл приложения и контроллеров
- Git: инициализация репозитория, коммиты, ветки

**Маршрут:**
- Переходишь с консоли на экран — нужно научиться создавать интерфейс. Изучаешь UIViewController как контейнер экрана, UIView как элементы интерфейса. Учишься выстраивать иерархию представлений. После этого шага ты выводишь текст, кнопки, поля ввода на экран.
- Интерфейс нужно оживить — нужна навигация между экранами. Изучаешь UINavigationController, переходы между контроллерами, передачу данных между экранами. После этого шага приложение имеет несколько экранов которые связаны между собой.
- Пользователь взаимодействует с приложением — нужно обрабатывать события. Изучаешь UIButton, UITextField, жесты, делегаты (delegates). Учишься реагировать на нажатия и ввод текста. После этого шага приложение отвечает на действия пользователя.
- Данные нужно отображать списками — изучаешь UITableView, UITableViewDataSource, UITableViewDelegate. Учишься заполнять таблицы данными, обновлять их. После этого шага ты выводишь большие наборы данных в удобном формате.
- Код нужно версионировать — инициализируешь Git репозиторий, делаешь коммиты, создаёшь ветки для фич. Учишься писать понятные сообщения коммитов. После этого шага код сохраняется и отслеживается, готов к публикации на GitHub.

**Milestone:** Могу создать iOS приложение с несколькими экранами, таблицей данных, обработкой ввода пользователя и версионированием в Git.


**Практика:**
- **Приложение для управления задачами** — Экран со списком задач (UITableView), кнопка добавления, экран создания новой задачи (UITextField), возможность отметить задачу как выполненную. Используешь UIViewController, UITableView, навигацию, Git для версионирования.
- **Каталог фильмов** — Главный экран со списком фильмов (UITableView), экран деталей фильма с описанием и рейтингом, поиск по названию (UITextField). Используешь UITableView, UIViewController, передачу данных между экранами, Git.
- **Приложение для заметок** — Список заметок (UITableView), экран создания/редактирования заметки с текстовым полем, удаление заметок свайпом. Используешь UITableView, UITextView, делегаты, навигацию, Git.

### Фаза 3: Portfolio (30 дней)

**Темы:**
- Юнит-тестирование: XCTest, тестирование моделей и логики
- Документирование кода: комментарии, README, архитектурные решения
- Публикация на GitHub: .gitignore, лицензия, структура репозитория
- Оптимизация интерфейса: адаптация к разным размерам экранов, обработка ошибок
- Деплой на устройство: сертификаты, provisioning profiles, тестирование на реальном iPhone

**Маршрут:**
- Приложение работает — нужно убедиться что оно работает правильно. Добавляешь юнит-тесты для моделей данных и функций обработки. Учишься писать тесты которые проверяют корректность логики. После этого шага есть тесты которые ловят ошибки при изменении кода.
- Код нужно объяснить — пишешь README с описанием приложения, архитектурой, инструкциями по запуску. Добавляешь комментарии к сложным функциям. После этого шага другой разработчик может понять и запустить твой проект.
- Проект готов к публикации — настраиваешь GitHub репозиторий: добавляешь .gitignore, выбираешь лицензию, организуешь структуру папок. После этого шага проект выглядит профессионально и готов к code review.
- Приложение должно работать везде — адаптируешь интерфейс к разным размерам экранов (iPhone SE, iPhone 14 Pro Max), добавляешь обработку ошибок (сетевые ошибки, пустые данные). После этого шага приложение стабильно работает на разных устройствах.
- Приложение готово к использованию — настраиваешь сертификаты Apple, provisioning profiles, тестируешь на реальном iPhone. Учишься собирать релиз-версию приложения. После этого шага приложение можно установить на реальное устройство и показать работодателю.

**Milestone:** Могу показать работодателю готовое iOS приложение с тестами, документацией, чистым кодом и инструкциями по запуску на GitHub.


**Практика:**

- **Каталог фильмов (финальная версия)** — Готовое приложение: тесты для фильтрации и поиска, подробная документация, адаптивный дизайн, обработка сетевых ошибок, инструкции по запуску, GitHub репозиторий с лицензией и .gitignore, возможность установки на реальное устройство.
- **Приложение для заметок (финальная версия)** — Полнофункциональное приложение: юнит-тесты для сохранения и удаления заметок, README с примерами использования, поддержка разных размеров экранов, обработка граничных случаев, деплой на iPhone, GitHub с чистой структурой и документацией.

## Gap-анализ

### Quick Wins (2–4 недели)

- Основы Swift синтаксиса (переменные, типы, операторы) — 3-5 дней, даст уверенность в языке
- Первый UIViewController на экране — 5-7 дней, покажет как работает iOS разработка
- UITableView с данными — 7-10 дней, основной паттерн для отображения списков в iOS
- Git коммиты и GitHub репозиторий — 2-3 дня, необходимо для портфолио и работы в команде

### Long Term (3+ месяца)

- Async/await и работа с сетью (URLSession) — 2-3 месяца, нужно для загрузки данных с сервера, базовый навык middle-разработчика
- Core Data для локального хранилища — 3-4 месяца, требуется для сохранения данных между запусками приложения, важно для production-приложений
- SwiftUI и современный подход к UI — 4-5 месяцев, новый стандарт Apple, постепенно вытесняет UIKit, нужен для карьерного роста
- Combine для реактивного программирования — 5-6 месяцев, продвинутый паттерн для управления потоками данных, требуется для senior-позиций
- Objective-C для поддержки legacy кода — 6-12 месяцев, многие компании имеют старые проекты, полезно но не критично для новичка

## Портфолио-проект

### Приложение для управления финансами — трекер расходов

**Проблема:** Пользователь не может отследить куда уходят деньги, нет понимания структуры расходов по категориям, сложно планировать бюджет без данных о прошлых тратах

**Сценарии использования:**
- Как пользователь я хочу добавить новый расход (сумма, категория, дата, описание) чтобы отследить свои траты
- Как пользователь я хочу увидеть список всех расходов в виде таблицы отсортированной по дате чтобы понять когда я потратил деньги
- Как пользователь я хочу увидеть статистику расходов по категориям (сколько потрачено на еду, транспорт, развлечения) чтобы понять где уходит больше всего денег
- Как пользователь я хочу удалить расход свайпом чтобы исправить ошибку
- Как пользователь я хочу отредактировать существующий расход чтобы исправить неправильно введённые данные

**Технические задачи:**
- UITableView с динамическим контентом — нужно научиться заполнять таблицу данными из массива, обновлять при добавлении/удалении, реализовать свайп для удаления
- Навигация между экранами — главный экран со списком, экран добавления расхода, экран редактирования, передача данных между контроллерами через segue
- Группировка и фильтрация данных — подсчёт расходов по категориям, сортировка по дате, вычисление статистики через функции и коллекции
- Сохранение данных между запусками — использование UserDefaults или простого JSON для сохранения расходов (без Core Data на этом этапе)
- Обработка ошибок и граничных случаев — пустой список, некорректный ввод суммы, отсутствие выбранной категории

**Навыки в проекте:**
- Swift: структуры для моделирования расхода (Expense), массивы для хранения, функции для группировки по категориям, замыкания для сортировки
- UIKit: UITableViewController для отображения списка, UIViewController для форм добавления/редактирования, UIPickerView для выбора категории, UITextField для ввода суммы
- Архитектура: разделение логики (модели) от представления (контроллеры), делегаты для обработки событий таблицы
- Git: инициализация репозитория, коммиты при добавлении каждой фичи, ветки для экспериментов, публикация на GitHub
- Тестирование: юнит-тесты для функций подсчёта расходов по категориям, проверка корректности группировки
- Документирование: README с описанием приложения, скриншоты экранов, инструкции по запуску, архитектурная диаграмма
