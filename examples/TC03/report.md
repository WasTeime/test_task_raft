## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Swift синтаксис и типы данных
- UIKit основы
- Навигация и жизненный цикл ViewController
- Auto Layout и constraints
- Работа с данными в памяти

**Маршрут:**
- Swift синтаксис → пишешь функции с параметрами, обрабатываешь ошибки через try-catch, работаешь со строками и коллекциями
- UIKit основы → создаёшь экран с кнопками, текстовыми полями и лейблами, при нажатии кнопки меняется текст на экране
- Навигация → переходишь между двумя экранами через push, передаёшь данные между ViewController, возвращаешься назад
- Auto Layout → вёрстаешь интерфейс так чтобы он одинаково выглядел на iPhone 12 и iPhone 15 Pro Max в портрете и ландшафте
- Работа с данными → сохраняешь массив объектов в UserDefaults, закрываешь приложение, открываешь — данные на месте

**Milestone:** Могу создать экран с формой, сохранить данные в UserDefaults и отобразить их на другом экране


**Практика:**
- **Калькулятор с историей** — Создаёшь интерфейс с кнопками цифр и операций, при нажатии отображается результат, история последних 10 операций сохраняется в UserDefaults и выводится в отдельном списке
- **Список дел с переходами** — Первый экран — список задач, при нажатии на задачу переходишь на второй экран с деталями и кнопкой редактирования, изменения сохраняются в памяти приложения
- **Адаптивный профиль пользователя** — Экран профиля с фото, именем, описанием и кнопками, вёрстаешь через Auto Layout так чтобы корректно отображалось на разных размерах экранов и при повороте устройства

### Фаза 2: Practice (30 дней)

**Темы:**
- Core Data для персистентности
- SwiftUI основы
- Combine и реактивное программирование
- Сетевые запросы и JSON
- Тестирование в Xcode

**Маршрут:**
- Core Data → создаёшь Entity для заметок, добавляешь запись через NSManagedObjectContext, закрываешь приложение, открываешь — запись на месте, фильтруешь по дате и категории
- SwiftUI основы → переписываешь один из UIKit проектов на SwiftUI, используешь @State для управления состоянием, @Binding для передачи данных между View
- Combine → подписываешься на изменения @Published свойства, обновляешь UI при изменении данных, используешь map и filter для трансформации потока данных
- Сетевые запросы → загружаешь список фильмов с публичного API (например TMDB), парсишь JSON, отображаешь в таблице с изображениями и описанием
- Тестирование → пишешь unit-тесты для функций обработки данных, интеграционные тесты для Core Data операций, запускаешь тесты в Xcode

**Milestone:** Могу создать приложение со списком данных из API, сохранить их в Core Data, отобразить с фильтрацией и написать тесты для основной логики


**Практика:**
- **Приложение погоды с сохранением** — Загружаешь погоду для города через OpenWeatherMap API, отображаешь температуру и прогноз на неделю, сохраняешь последние 5 запросов в Core Data, можешь переключаться между ними без интернета
- **Каталог книг с фильтрацией** — Загружаешь список книг с Google Books API, отображаешь в таблице с обложками, фильтруешь по жанру и автору через Combine, добавляешь в избранное (сохраняется в Core Data), пишешь тесты для фильтрации
- **Переписка UIKit проекта на SwiftUI** — Берёшь один из проектов Foundation фазы, переписываешь на SwiftUI с @State и @Binding, добавляешь Combine для реактивности, сохраняешь данные в Core Data вместо UserDefaults

### Фаза 3: Portfolio (30 дней)

**Темы:**
- Архитектура MVVM и Coordinator
- Продвинутая работа с Core Data
- Оптимизация производительности
- Публикация в App Store
- CI/CD и TestFlight

**Маршрут:**
- MVVM архитектура → рефакторишь проект с чистым разделением View, ViewModel и Model, ViewModel содержит всю бизнес-логику, View только отображает данные
- Coordinator паттерн → реализуешь навигацию через Coordinator, избегаешь циклических зависимостей между ViewController, легко добавляешь новые экраны
- Core Data оптимизация → используешь NSFetchedResultsController для больших объёмов данных, индексируешь часто используемые поля, профилируешь через Instruments
- Подготовка к App Store → создаёшь App Store Connect запись, готовишь скриншоты и описание, настраиваешь сертификаты и provisioning profiles в Xcode
- TestFlight и CI/CD → загружаешь сборку на TestFlight через Xcode, настраиваешь GitHub Actions для автоматической сборки при push в main ветку

**Milestone:** Могу опубликовать приложение в App Store с чистой MVVM архитектурой, оптимизированной Core Data и автоматической сборкой через CI/CD


**Практика:**
- **Рефакторинг проекта на MVVM** — Берёшь один из предыдущих проектов, переписываешь на MVVM с отдельными ViewModel классами, добавляешь Coordinator для навигации, профилируешь через Instruments и оптимизируешь запросы к Core Data
- **Приложение с большим объёмом данных** — Создаёшь приложение со списком 10000+ записей (например контакты или товары), используешь NSFetchedResultsController для пагинации, индексируешь поля, профилируешь память через Instruments, список остаётся отзывчивым
- **Подготовка к публикации** — Берёшь готовый проект, создаёшь App Store Connect запись, готовишь скриншоты и описание, загружаешь на TestFlight, настраиваешь GitHub Actions для автоматической сборки и отправки на TestFlight при push

## Портфолио-проект

### FinanceTracker — приложение для управления личными финансами

**Проблема:** Человек не видит куда уходят его деньги, не может отследить расходы по категориям, не планирует бюджет и не понимает свои финансовые привычки

**Сценарии использования:**
- Открываю приложение → вижу сегодняшние расходы и остаток бюджета на месяц
- Нажимаю кнопку добавить расход → выбираю категорию, сумму, дату, описание → расход сохраняется и появляется в списке
- Свайпаю расход влево → удаляю его, данные обновляются в реальном времени
- Переходу на вкладку статистика → вижу круговую диаграмму расходов по категориям за месяц, могу переключаться между месяцами
- Устанавливаю бюджет на месяц → приложение показывает прогресс заполнения бюджета, предупреждает если превышу лимит
- Закрываю приложение → открываю через неделю → все мои расходы на месте, статистика актуальна

**Технические задачи:**
- Core Data — сохранение тысяч транзакций, быстрая фильтрация по дате и категории, индексирование для оптимальной производительности при работе с большим объёмом данных
- SwiftUI и Combine — реактивное обновление UI при добавлении/удалении расходов, синхронизация данных между экранами через @Published, анимация диаграмм при изменении данных
- Архитектура MVVM — разделение логики расчёта статистики, фильтрации и валидации в ViewModel, чистое разделение ответственности между слоями
- Производительность — работа с большим количеством записей без лагов, оптимизация запросов к Core Data, профилирование через Instruments

**Навыки в проекте:**
- SwiftUI — создание адаптивного интерфейса с TabView, List, Form, кастомными View компонентами
- Core Data — создание Entity для транзакций и категорий, CRUD операции, NSFetchedResultsController для пагинации больших объёмов данных
- Combine — @Published для реактивности, подписки на изменения данных, трансформация потоков через map и filter
- MVVM архитектура — ViewModel с бизнес-логикой расчёта статистики и фильтрации, чистое разделение от View
- Тестирование — unit-тесты для расчёта статистики, интеграционные тесты для Core Data операций
- Графики и визуализация — использование Charts библиотеки для круговой диаграммы расходов по категориям
- Локализация и форматирование — форматирование денежных сумм, локализация дат и категорий для разных локалей
