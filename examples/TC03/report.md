## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Swift синтаксис и типы данных
- Основы SwiftUI
- Навигация и состояние
- Работа с сетью (URLSession)
- Git и GitHub

**Маршрут:**
- Swift синтаксис → пишешь функции с параметрами, используешь массивы и словари, обрабатываешь ошибки через try-catch
- SwiftUI View → рисуешь кнопки, текстовые поля, списки; при нажатии кнопки меняется состояние и обновляется экран
- State и Binding → создаёшь счётчик: нажимаешь кнопку, число увеличивается, значение сохраняется в памяти приложения
- URLSession → загружаешь JSON с публичного API (например, JSONPlaceholder), парсишь данные в структуру Swift, выводишь на экран
- Git базовый → создаёшь репозиторий, коммитишь изменения, пушишь на GitHub, создаёшь pull request

**Milestone:** Могу создать простое приложение со списком элементов, загруженных с API, с сохранением состояния при навигации


**Практика:**
- **Счётчик с историей** — Создаёшь приложение с кнопками +1 и -1, текущее значение отображается на экране. При нажатии кнопки 'История' видишь список всех изменений (время и значение). Данные сохраняются в памяти приложения во время сессии
- **Список публичных постов** — Загружаешь посты с JSONPlaceholder API, отображаешь их в списке (заголовок и текст). При нажатии на пост видишь полный текст на отдельном экране. Есть кнопка 'Обновить' для перезагрузки данных
- **Конвертер валют** — Вводишь сумму в одной валюте, выбираешь целевую валюту из выпадающего списка, видишь результат. Используешь публичный API для курсов (например, exchangerate-api.com). Результат обновляется при изменении суммы или валюты

### Фаза 2: Practice (30 дней)

**Темы:**
- Продвинутый SwiftUI (MVVM, ObservableObject)
- Combine для реактивности
- CoreData для локального хранилища
- Обработка ошибок и сетевые запросы
- Unit-тестирование
- GitHub Actions для CI/CD

**Маршрут:**
- MVVM архитектура → разделяешь код на View, ViewModel и Model; ViewModel управляет состоянием и логикой, View только отображает
- ObservableObject и @Published → создаёшь ViewModel с @Published свойствами, View автоматически обновляется при изменении данных
- Combine Publisher → используешь .map, .filter, .debounce для трансформации данных; подписываешься на изменения и обновляешь UI
- CoreData миграция → создаёшь Entity для сохранения данных, пишешь функции для добавления, удаления, фильтрации записей; данные сохраняются после закрытия приложения
- XCTest → пишешь тесты для ViewModel (проверяешь логику фильтрации, валидации), запускаешь тесты в Xcode
- GitHub Actions → настраиваешь workflow для автоматического запуска тестов при push, видишь результаты в GitHub

**Milestone:** Могу создать приложение с архитектурой MVVM, локальным хранилищем данных, реактивными обновлениями UI и покрытием unit-тестами


**Практика:**
- **Приложение для управления задачами** — Создаёшь список задач с возможностью добавления, удаления и отметки как выполненной. Используешь CoreData для сохранения. Фильтруешь по статусу (все, активные, выполненные). Данные сохраняются после закрытия приложения. Пишешь тесты для логики фильтрации
- **Поиск фильмов с кешированием** — Вводишь название фильма, загружаешь результаты с OMDB API (или аналога), отображаешь постеры и описания. Используешь Combine для debounce поиска (не каждый символ отправляет запрос). Кешируешь результаты в CoreData. При повторном поиске видишь кешированные данные
- **Трекер привычек** — Добавляешь привычку (название, цель в днях), отмечаешь дни выполнения. Видишь прогресс в процентах и график полос (зелёные дни выполнения, серые пропуски). Данные сохраняются в CoreData. Используешь MVVM для управления состоянием. Настраиваешь GitHub Actions для запуска тестов

## Портфолио-проект

### FinanceTracker — приложение для управления личными финансами

**Проблема:** Человек не видит куда уходят его деньги, не может выявить паттерны расходов и спланировать бюджет. Существующие приложения либо слишком сложные, либо не сохраняют данные локально и требуют интернета

**Сценарии использования:**
- Открываю приложение → вижу сегодняшние расходы и остаток бюджета на месяц
- Нажимаю '+' → добавляю расход (сумма, категория, описание, дата) → данные сохраняются и обновляют остаток
- Открываю вкладку 'Статистика' → вижу pie chart расходов по категориям за месяц, могу переключаться между месяцами
- Ищу расход по названию → список фильтруется в реальном времени, результаты подсвечиваются
- Удаляю расход → он исчезает из списка и статистики, остаток бюджета пересчитывается

**Технические задачи:**
- CoreData — сохранение и фильтрация тысяч записей расходов по дате, категории и сумме без тормозов при прокрутке списка
- Combine и реактивность — при добавлении расхода автоматически обновляются pie chart, остаток бюджета и список без перезагрузки экрана
- Оптимизация производительности — отрисовка pie chart с 100+ записями, ленивая загрузка списков, кеширование вычисленных значений
- Локализация — поддержка русского и английского языков для названий категорий, валют, интерфейса
- Доступность — VoiceOver поддержка для навигации по категориям, чтения сумм, взаимодействия с графиками

**Навыки в проекте:**
- SwiftUI и MVVM — структура приложения с ViewModel для управления состоянием расходов и фильтрации
- CoreData — создание Entity для расходов, миграция схемы, запросы с NSPredicate для фильтрации по дате и категории
- Combine — использование @Published для реактивного обновления UI при изменении данных, debounce для поиска
- Графики и визуализация — рисование pie chart с использованием Canvas или сторонней библиотеки (Charts)
- Тестирование — unit-тесты для ViewModel (проверка расчётов, фильтрации), интеграционные тесты для CoreData
- Git и GitHub — версионирование кода, pull request, документирование в README
- Локализация и доступность — Localizable.strings для переводов, VoiceOver метки для элементов UI
- TestFlight — подготовка приложения к бета-тестированию и публикации в App Store
