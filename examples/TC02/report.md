**Темы:**
- Python для ML (NumPy, Pandas)
- SQL для работы с данными
- Основы машинного обучения
- Git и версионирование

**Маршрут:**
- NumPy → создаёшь матрицы, выполняешь линейную алгебру, вычисляешь скалярное произведение и норму вектора
- Pandas → загружаешь CSV с 100к строк, находишь пропуски, заполняешь медианой, группируешь по категориям и строишь сводные таблицы
- SQL → пишешь SELECT с JOIN, WHERE, GROUP BY, находишь топ-10 клиентов по сумме покупок, агрегируешь данные по датам
- Scikit-learn → тренируешь линейную регрессию на датасете, вычисляешь MSE и R², делаешь train-test split 80-20
- Git → инициализируешь репозиторий, коммитишь изменения, создаёшь ветки, мержишь в main

**Milestone:** Могу загрузить CSV, очистить данные, обучить простую модель линейной регрессии и оценить её качество на тестовом наборе


**Практика:**
- **Анализ датасета Iris** — Загружаешь датасет Iris, исследуешь распределение признаков, находишь корреляции между ними, строишь гистограммы и scatter plots, вычисляешь статистику по видам цветов
- **Предсказание цены дома** — Берёшь датасет с ценами домов, заполняешь пропуски медианой, нормализуешь признаки, тренируешь линейную регрессию, вычисляешь MSE на тесте, сравниваешь с базовой моделью
- **SQL-анализ продаж** — Пишешь SQL-запросы к базе продаж: топ-10 товаров по выручке, среднее значение покупки по месяцам, клиенты с суммой покупок выше медианы, группировка по категориям и регионам

## Портфолио-проект

### Система предсказания оттока клиентов — Production ML Pipeline

**Проблема:** Компания теряет 15-20% клиентов ежегодно и не может предсказать кто уйдёт. Нет системы для раннего выявления риска, поэтому нельзя принять профилактические меры. Бизнес теряет миллионы на привлечение новых клиентов вместо удержания существующих.

**Сценарии использования:**
- Я — аналитик: загружаю CSV с историей клиентов, система обучается на 80% данных, показывает метрики качества (precision, recall, AUC) на тестовом наборе
- Я — бизнес-пользователь: вижу список клиентов с риском оттока (вероятность > 70%), сортирую по приоритету, экспортирую в Excel для отдела продаж
- Я — DevOps: деплою модель в Kubernetes, настраиваю автоматическое переобучение каждую неделю, мониторю точность модели и алерты при дрифте
- Я — ML-инженер: логирую все эксперименты в MLflow, версионирую датасеты, сравниваю 3 модели (логистическая регрессия, XGBoost, нейросеть), выбираю лучшую

**Технические задачи:**
- Дисбаланс классов — 95% клиентов не уходят, 5% уходят. Используешь SMOTE для балансировки, подбираешь threshold вероятности, оптимизируешь по recall вместо accuracy
- Масштабируемость — датасет растёт с 100k до 1M записей. Используешь Pandas для подготовки, PyTorch DataLoader для батчей, распределённое обучение на нескольких GPU
- Production-ready pipeline — модель должна переобучаться автоматически, логировать метрики, отправлять алерты при падении качества. Используешь MLflow для логирования, Kubernetes CronJob для переобучения, Prometheus для мониторинга

**Навыки в проекте:**
- Python (Pandas, NumPy) — подготовка данных: заполнение пропусков, кодирование категорических признаков, нормализация
- SQL — извлечение данных из базы: история покупок, взаимодействия с поддержкой, платежи клиентов
- Scikit-learn и XGBoost — обучение моделей: логистическая регрессия как baseline, XGBoost для лучшей точности, кросс-валидация
- PyTorch — нейросеть для сравнения: полносвязная сеть с 3 слоями, dropout для регуляризации, обучение с Adam оптимизатором
- Docker — упаковка приложения: Dockerfile с Python, FastAPI для API, requirements.txt с зависимостями
- Kubernetes — деплой: YAML для deployment, Service для доступа, CronJob для переобучения каждую неделю
- AWS — облачная инфраструктура: S3 для хранения датасетов и моделей, EC2 для обучения, IAM для доступа
- Git — версионирование: коммиты с описанием экспериментов, ветки для разных моделей, pull requests для code review
- MLflow — отслеживание экспериментов: логирование параметров, метрик, артефактов, сравнение моделей в UI
- Коммуникация — презентация результатов: объяснение бизнесу что такое precision и recall, почему модель выбрана, какой ROI ожидается
