## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Python для ML (NumPy, Pandas)
- SQL для анализа данных
- Основы машинного обучения
- PyTorch базовые операции
- Docker для воспроизводимости

**Маршрут:**
- NumPy → создаёшь матрицы, выполняешь операции линейной алгебры, вычисляешь градиенты вручную
- Pandas → загружаешь CSV с 100k строк, чистишь пропуски, группируешь по категориям, строишь сводные таблицы
- SQL → пишешь запросы с JOIN, GROUP BY, агрегирующие функции, фильтруешь по условиям на датасете 1M+ записей
- PyTorch тензоры → создаёшь тензоры, выполняешь операции, вычисляешь градиенты через autograd, видишь как меняются веса
- Docker → пакуешь Python-скрипт с зависимостями в контейнер, запускаешь на другой машине, результат идентичен

**Milestone:** Могу загрузить датасет, очистить пропуски, создать простую модель линейной регрессии на PyTorch, упаковать в Docker


**Практика:**
- **Анализ продаж с Pandas и SQL** — Загружаешь CSV с продажами, чистишь дубликаты и пропуски, группируешь по месяцам и категориям, пишешь SQL-запросы для поиска топ-товаров, строишь графики тренда продаж
- **Линейная регрессия на PyTorch** — Создаёшь датасет синтетических данных (X, y), инициализируешь веса, пишешь цикл обучения с MSE-лоссом, видишь как лосс падает, делаешь предсказания на новых данных
- **Docker-контейнер для ML-скрипта** — Пишешь Python-скрипт для обработки данных, создаёшь Dockerfile с зависимостями (numpy, pandas), собираешь образ, запускаешь контейнер, проверяешь что результаты воспроизводятся

## Портфолио-проект

### SmartDocAssistant — RAG-система для анализа корпоративных документов

**Проблема:** Сотрудники компании теряют часы на поиск информации в сотнях документов (контракты, политики, отчёты). Нет единого способа задать вопрос и получить ответ с ссылкой на источник. Информация разрозненна и неструктурирована.

**Сценарии использования:**
- Загружаю PDF-документы в систему → система создаёт embeddings и индексирует их в векторной БД
- Задаю вопрос на русском языке → система находит релевантные фрагменты и возвращает ответ с цитатами и ссылками на документы
- Вижу время обработки запроса и количество найденных источников → понимаю надёжность ответа
- Система работает 24/7 в облаке → я могу обращаться к ней с любого устройства через веб-интерфейс

**Технические задачи:**
- LangChain + Vector DB (Pinecone/Weaviate) — индексирование 10k+ документов, поиск релевантных фрагментов за <1 сек, обработка многоязычных запросов
- PyTorch embedding модель (или OpenAI API) — преобразование текста в векторы, обеспечение качества поиска через fine-tuning на корпоративных данных
- Docker + Kubernetes + AWS — упаковка сервиса, масштабирование под нагрузку, кэширование результатов, мониторинг latency и accuracy ответов

**Навыки в проекте:**
- LangChain — создание цепочки Retriever → LLM, обработка контекста, форматирование ответов с источниками
- PyTorch — использование pre-trained embedding моделей (Sentence-Transformers), возможность fine-tuning на специфичных данных
- SQL + Pandas — обработка метаданных документов, логирование запросов и ответов для анализа
- Docker + Docker Compose — контейнеризация ML-сервиса, Redis для кэширования, PostgreSQL для хранения метаданных
- Kubernetes — развёртывание на кластере, автоскейлинг по нагрузке, health checks и graceful shutdown
- AWS SageMaker / EC2 — хостинг модели, создание API endpoint, настройка мониторинга через CloudWatch
- MLflow — логирование версий моделей, параметров retriever, метрик качества ответов (BLEU, ROUGE)
- Коммуникация с бизнесом — демонстрация ROI (сокращение времени поиска на 80%), сбор feedback от пользователей для улучшения
