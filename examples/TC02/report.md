## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Синтаксис Python и типы данных
- Функции, модули и организация кода
- Работа с файлами и обработка данных
- Основы SQL и запросы к базам данных
- Структуры данных: списки, словари, массивы

**Маршрут:**
- Начинаешь с синтаксиса Python — без этого нельзя написать ни одну программу. Пишешь простые скрипты: переменные, условия, циклы. После этого шага ты выполняешь базовые вычисления и обрабатываешь текст.
- Скрипты растут — нужно их организовать. Учишься писать функции и разбивать код на модули. После этого шага ты переиспользуешь код и избегаешь дублирования.
- Программы работают с данными — нужно их структурировать. Работаешь со списками, словарями, кортежами. После этого шага ты группируешь и фильтруешь информацию в памяти.
- Данные нужно сохранять и читать. Учишься работать с файлами (CSV, JSON, текст). После этого шага ты загружаешь датасеты и сохраняешь результаты анализа.
- Большие данные хранятся в базах. Учишься писать SQL-запросы: SELECT, WHERE, JOIN, агрегация. После этого шага ты извлекаешь нужные данные из реальных БД для анализа.

**Milestone:** Могу написать скрипт на Python, который читает CSV-файл, фильтрует данные по условию и сохраняет результат в новый файл. Могу написать SQL-запрос с WHERE и JOIN для извлечения данных из таблицы.


**Практика:**
- **Анализ продаж интернет-магазина** — Пишешь скрипт на Python, который читает CSV с заказами (дата, товар, цена, регион). Фильтруешь заказы по периоду, считаешь сумму по регионам, сохраняешь топ-5 регионов в новый файл. Результат: таблица с суммами продаж по регионам.
- **Обработка логов веб-сервера** — Пишешь скрипт, который парсит текстовый лог (IP, время, статус, URL). Считаешь количество ошибок (статус 4xx, 5xx), находишь самый частый URL, выводишь статистику. Результат: отчёт с ошибками и популярными страницами.
- **Анализ данных о студентах** — Пишешь скрипт, который читает JSON с информацией о студентах (имя, возраст, оценки, курс). Группируешь по курсам, считаешь среднюю оценку, находишь студентов с оценкой выше среднего. Результат: список студентов по курсам с их средними оценками.

## Портфолио-проект

### ML Pipeline для предсказания оттока клиентов — от данных к продакшену

**Проблема:** Компания теряет клиентов, но не знает кто уйдёт. Нужна модель, которая предсказывает отток, чтобы отдел продаж мог удержать рискующих клиентов.

**Сценарии использования:**
- Я аналитик: загружу CSV с данными о клиентах, запущу скрипт и получу список клиентов с вероятностью оттока
- Я менеджер: вижу в отчёте, какие факторы влияют на отток (возраст, расходы, длительность подписки), чтобы принять решение
- Я разработчик: загружу сохранённую модель и получу предсказание для одного клиента через функцию predict()

**Технические задачи:**
- Обработка пропусков и категориальных признаков в Pandas — нужно правильно подготовить данные, чтобы модель работала
- Дисбаланс классов (мало клиентов с оттоком) — нужно выбрать правильную метрику (не просто accuracy, а ROC-AUC или F1)
- Воспроизводимость результатов — нужно зафиксировать seed и версию данных, чтобы коллега получил те же результаты

**Навыки в проекте:**
- Pandas: загрузка CSV, удаление пропусков, кодирование категорий, создание новых признаков
- Логистическая регрессия: обучение модели, предсказание вероятностей, оценка качества (ROC-AUC, F1)
- Визуализация: графики распределения признаков, ROC-кривая, матрица ошибок
- Git и GitHub: версионирование кода, README с инструкциями, коммиты с описаниями
- Тестирование: unit-тесты для функций очистки данных и предсказания
- Документация: README, комментарии в коде, конфиг с параметрами, финальный отчёт с выводами
