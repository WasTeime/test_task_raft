## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Синтаксис Python и типы данных
- Функции и модули
- Структуры данных (списки, словари, кортежи)
- Работа с файлами и исключения
- Основы SQL и реляционные базы данных

**Маршрут:**
- Начинаешь с синтаксиса Python — без этого нельзя написать ни одной программы. Пишешь простые скрипты: переменные, условия, циклы. После этого шага ты выполняешь базовые вычисления и обрабатываешь текст.
- Скрипты растут — нужно их организовать. Учишься писать функции и разбивать код на модули. После этого шага ты переиспользуешь код и делаешь программы понятнее.
- Программы работают с данными — нужно их структурировать. Работаешь со списками, словарями, кортежами, множествами. После этого шага ты фильтруешь, группируешь и трансформируешь данные в памяти.
- Данные нужно сохранять между запусками. Учишься читать и писать файлы, обрабатывать ошибки через исключения. После этого шага ты создаёшь программы которые работают с реальными файлами и не падают на ошибках.
- Данные нужно хранить структурированно. Изучаешь SQL: SELECT, INSERT, UPDATE, DELETE, JOIN, агрегирующие функции. После этого шага ты пишешь запросы к базам данных и понимаешь как организованы данные в реальных приложениях.

**Milestone:** Могу написать скрипт на Python который читает CSV-файл, фильтрует данные по условию и записывает результат в новый файл. Могу написать SQL-запрос с JOIN и WHERE для выборки данных из нескольких таблиц.


**Практика:**
- **Анализатор логов веб-сервера** — Пишешь скрипт который читает лог-файл, парсит строки, фильтрует ошибки по коду (404, 500), группирует по IP-адресам и выводит статистику. Результат: текстовый отчёт с топ-10 IP которые генерировали ошибки.
- **Калькулятор расходов на проекты** — Пишешь скрипт который читает CSV с расходами (дата, категория, сумма), фильтрует по периоду, суммирует по категориям, вычисляет среднее. Результат: таблица с итогами по категориям и общей суммой.
- **Парсер данных о товарах** — Пишешь скрипт который читает текстовый файл с описаниями товаров, извлекает цену и название, фильтрует по диапазону цен, сортирует. Результат: отформатированный список товаров в нужном диапазоне.

### Фаза 2: Practice (30 дней)

**Темы:**
- Django: модели, ORM и миграции
- Django: представления (views) и маршруты (urls)
- Django: формы и валидация данных
- PostgreSQL и работа с реальной БД
- Git: версионирование и совместная работа

**Маршрут:**
- Теперь пишешь не скрипты а веб-приложение. Изучаешь Django: как устроены модели (классы которые описывают таблицы), как работает ORM (пишешь Python вместо SQL), как создавать миграции (изменения структуры БД). После этого шага ты создаёшь структуру приложения и таблицы в БД через Python-код.
- Приложение должно отвечать на запросы. Учишься писать views (функции которые обрабатывают запросы) и urls (маршруты которые связывают URL с views). После этого шага ты создаёшь API-эндпоинты которые возвращают данные из БД.
- Данные приходят от пользователя — нужно их проверить. Изучаешь Django Forms: как описывать форму, как валидировать данные, как обрабатывать ошибки. После этого шага ты принимаешь данные от клиента, проверяешь их и сохраняешь в БД.
- Переходишь с SQLite на PostgreSQL — настоящую БД для production. Учишься подключать PostgreSQL к Django, работать с более сложными запросами, понимать индексы и производительность. После этого шага ты работаешь с реальной БД которая может обслуживать много пользователей.
- Код нужно версионировать и делиться с командой. Изучаешь Git: создание репозиториев, коммиты, ветки, pull requests. После этого шага ты можешь работать в команде, делиться кодом и видеть историю изменений.

**Milestone:** Могу создать Django-приложение с моделями, views и формами. Могу написать эндпоинт который принимает данные, валидирует их, сохраняет в PostgreSQL и возвращает результат. Могу создать репозиторий на GitHub и сделать pull request.


**Практика:**
- **API для управления списком задач** — Создаёшь Django-приложение с моделью Task (название, описание, статус, дата создания). Пишешь views для создания, чтения, обновления и удаления задач. Добавляешь форму для валидации. Подключаешь PostgreSQL. Результат: рабочий API который можно тестировать через curl или Postman.
- **Система управления контактами** — Создаёшь Django-приложение с моделью Contact (имя, email, телефон, компания). Пишешь views для CRUD операций. Добавляешь валидацию email и телефона. Реализуешь фильтрацию по компании. Результат: приложение которое хранит контакты в PostgreSQL и позволяет их искать.
- **API для каталога книг** — Создаёшь Django-приложение с моделями Book (название, автор, год, жанр) и Author. Пишешь views для получения книг по жанру и автору. Добавляешь форму для добавления новой книги. Результат: приложение которое управляет каталогом и позволяет фильтровать книги.

### Фаза 3: Portfolio (30 дней)

**Темы:**
- Тестирование: unit-тесты и интеграционные тесты
- Docker: контейнеризация приложения

- Деплой приложения и CI/CD основы
- Code review и best practices

**Маршрут:**
- Код должен быть надёжным. Учишься писать тесты: unit-тесты для функций, интеграционные тесты для views. Пишешь тесты для моделей, views и форм. После этого шага ты уверен что приложение работает правильно и не сломается при изменениях.
- Приложение нужно запустить везде одинаково. Изучаешь Docker: как создавать образы, как писать Dockerfile, как запускать контейнеры. После этого шага ты упаковываешь приложение в контейнер который работает везде: на твоём компьютере, на сервере, в облаке.
- Код нужно документировать. Пишешь docstrings для функций и классов, создаёшь README с инструкциями по установке и использованию, описываешь API. После этого шага другой разработчик может понять твой код и запустить приложение.

**Milestone:** Могу написать тесты которые покрывают 70% кода. Могу создать Dockerfile и запустить приложение в контейнере. Могу создать README с полной инструкцией. Могу задеплоить приложение в интернет. Мой код готов к code review в команде.

## Портфолио-проект

### API для управления проектами и задачами — платформа для команд

**Проблема:** Команды разработчиков не имеют простого способа отслеживать задачи, распределять работу и видеть прогресс проекта. Нужно приложение которое позволяет создавать проекты, добавлять задачи, назначать их на людей и отслеживать статус.

**Сценарии использования:**
- Как менеджер я могу создать новый проект и добавить описание, чтобы команда знала что нужно делать.
- Как разработчик я могу создать задачу в проекте, указать приоритет и дедлайн, чтобы организовать работу.
- Как менеджер я могу назначить задачу на разработчика и изменить статус (новая, в работе, готово), чтобы отслеживать прогресс.
- Как разработчик я могу получить список моих задач отфильтрованный по статусу и приоритету, чтобы знать что делать в первую очередь.
- Как менеджер я могу получить отчёт по проекту: сколько задач готово, сколько в работе, сколько просрочено, чтобы видеть статус проекта.

**Технические задачи:**
- Модели и связи: нужно создать модели Project, Task, User с правильными связями (один проект — много задач, одна задача — один исполнитель). Сложность: правильно настроить foreign keys и миграции.
- Фильтрация и поиск: нужно реализовать фильтрацию задач по статусу, приоритету, исполнителю и поиск по названию. Сложность: написать эффективные SQL-запросы через ORM.
- Валидация данных: нужно проверить что дедлайн не в прошлом, приоритет из допустимого набора, статус может переходить только в определённые состояния. Сложность: написать кастомные валидаторы в Django Forms.
- Тестирование: нужно покрыть тестами создание проекта, добавление задачи, изменение статуса, фильтрацию. Сложность: написать интеграционные тесты которые проверяют весь flow.

**Навыки в проекте:**
- Django models: создание моделей Project, Task, User с полями и связями. Использование ForeignKey для связей между таблицами.
- Django ORM: написание запросов для фильтрации задач по нескольким условиям, использование select_related и prefetch_related для оптимизации.
- Django views и forms: создание views для CRUD операций, использование Forms для валидации данных при создании и обновлении задач.
- PostgreSQL: использование реальной БД, понимание индексов, написание миграций для изменения структуры.
- Git: версионирование кода, создание осмысленных коммитов, работа с ветками.
- Тестирование: написание unit-тестов для моделей и интеграционных тестов для views.
- Docker: контейнеризация приложения, создание Dockerfile и docker-compose.yml для локальной разработки.
- Документация: подробный README с инструкциями по установке, примеры API запросов, описание моделей и эндпоинтов.
- Деплой: развёртывание приложения на Heroku или простом VPS, настройка переменных окружения, работа с production БД.
