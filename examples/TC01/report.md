## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Django ORM и миграции
- REST API с Django REST Framework
- PostgreSQL и индексы
- Docker базовые концепции
- Аутентификация и авторизация

**Маршрут:**
- Django ORM → создаёшь модели User и Post, добавляешь записи через shell, фильтруешь по дате и автору, видишь SQL-запросы в логах
- Миграции → меняешь поле в модели, запускаешь migrate, откатываешь на предыдущую версию, видишь что данные сохранились
- Django REST Framework → endpoint GET /api/posts возвращает JSON со списком постов, POST /api/posts создаёт новый пост, DELETE удаляет
- PostgreSQL → создаёшь индекс на поле created_at, запрос по дате работает в 10 раз быстрее, видишь разницу в EXPLAIN ANALYZE
- Docker → пишешь Dockerfile для Django приложения, собираешь образ, запускаешь контейнер, приложение доступно на localhost:8000

**Milestone:** Могу создать Django приложение с моделями, REST API эндпоинтами, PostgreSQL базой и запустить в Docker контейнере


**Практика:**
- **Блог с комментариями** — Создаёшь модели Post и Comment, пишешь API для получения постов с фильтром по автору, добавления комментариев, удаления своих постов. Данные хранятся в PostgreSQL, приложение работает в Docker
- **Система управления задачами** — Создаёшь модели Task и User, пишешь эндпоинты для создания задач, изменения статуса, фильтрации по приоритету и дате. Добавляешь базовую аутентификацию через токены
- **API каталога товаров** — Создаёшь модели Product и Category, пишешь эндпоинты для получения товаров с пагинацией, фильтром по цене и категории, поиском по названию. Добавляешь индексы в PostgreSQL для оптимизации

### Фаза 2: Practice (30 дней)

**Темы:**
- FastAPI и асинхронность
- Тестирование (pytest, unittest)
- Кэширование и Redis
- Docker Compose и многоконтейнерные приложения
- CI/CD базовые концепции

**Маршрут:**
- FastAPI → создаёшь async функции, endpoint /api/items возвращает JSON, используешь Pydantic для валидации, видишь автоматическую документацию в Swagger
- Pytest → пишешь тесты для API эндпоинтов, проверяешь статус коды и структуру ответов, запускаешь coverage и видишь что покрыто на 85%
- Redis → кэшируешь результаты дорогих запросов, при повторном запросе данные приходят из кэша за 1мс вместо 500мс
- Docker Compose → описываешь в docker-compose.yml три сервиса (Django, PostgreSQL, Redis), запускаешь docker-compose up и всё работает вместе
- GitHub Actions → пишешь workflow который при push запускает тесты, проверяет линтер, собирает Docker образ и выкладывает на Docker Hub

**Milestone:** Могу написать FastAPI приложение с асинхронными функциями, покрыть тестами, добавить кэширование через Redis, запустить через Docker Compose и настроить CI/CD в GitHub Actions


**Практика:**
- **Микросервис рекомендаций** — Пишешь FastAPI приложение которое получает ID пользователя, возвращает рекомендованные товары. Результаты кэшируются в Redis на 1 час. Пишешь тесты для всех эндпоинтов. Запускаешь через Docker Compose с PostgreSQL и Redis
- **API аналитики с асинхронностью** — Создаёшь FastAPI приложение с async функциями для обработки событий, сохранения в PostgreSQL и кэширования статистики в Redis. Пишешь unit-тесты и интеграционные тесты. Настраиваешь GitHub Actions для автоматического запуска тестов
- **Многоконтейнерное приложение** — Пишешь Django приложение с FastAPI микросервисом, оба работают с одной PostgreSQL базой и Redis кэшем. Описываешь всё в docker-compose.yml, добавляешь nginx как reverse proxy, настраиваешь логирование

## Портфолио-проект

### SaaS платформа управления проектами — Collaborative Task Manager

**Проблема:** Команды теряют время на синхронизацию задач в разных инструментах, не видят реального прогресса проекта, не могут быстро масштабироваться при росте команды, теряют данные при сбоях серверов

**Сценарии использования:**
- Создаю проект → приглашаю членов команды → они видят все задачи в реальном времени
- Создаю задачу → назначаю исполнителя → он получает уведомление → меняет статус → я вижу обновление за 100мс
- Фильтрую задачи по приоритету, исполнителю, дате → вижу только релевантные → экспортирую в CSV
- Система обрабатывает 10000 одновременных пользователей → не падает → автоматически масштабируется
- Сервер упал → данные восстановились из бэкапа → потеря менее 1 минуты

**Технические задачи:**
- Django ORM с оптимизацией — фильтрация 100k задач по 5 критериям без N+1 запросов, индексы на PostgreSQL, кэширование в Redis
- WebSocket через Django Channels — реальное время обновления задач для 100+ пользователей в одном проекте без задержек
- Kubernetes и автоскейлинг — приложение масштабируется с 1 до 10 подов при росте нагрузки, сессии хранятся в Redis, не теряются при перезагрузке
- AWS RDS с Multi-AZ — автоматический failover при сбое, ежедневные бэкапы, восстановление за 5 минут
- Мониторинг и алерты — Prometheus метрики, Grafana дашборд, Slack уведомления при ошибках, SLA 99.9%

**Навыки в проекте:**
- Django ORM и миграции — модели Project, Task, User с связями, оптимизированные запросы с select_related и prefetch_related
- Django REST Framework — полный CRUD API с авторизацией через JWT, пагинация, фильтры, поиск
- PostgreSQL и индексы — индексы на часто фильтруемые поля, EXPLAIN ANALYZE для оптимизации, полнотекстовый поиск
- Redis и кэширование — кэширование списков задач, инвалидация при изменении, хранение сессий
- Docker и Docker Compose — Dockerfile для приложения, docker-compose.yml с Django, PostgreSQL, Redis, Nginx
- Kubernetes — deployment.yaml, service, ingress, horizontal pod autoscaler, health checks
- AWS — RDS для базы, S3 для файлов, CloudFront для кэширования, Application Load Balancer
- Мониторинг — Prometheus метрики (requests, errors, latency), Grafana дашборд, алерты в Slack
- CI/CD — GitHub Actions для запуска тестов, линтера, сборки Docker образа, деплоя на AWS
- Тестирование — pytest для unit-тестов, интеграционные тесты для API, coverage 85%+
