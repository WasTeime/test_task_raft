## План обучения (90 дней)

### Фаза 1: Foundation (30 дней)

**Темы:**
- Python: синтаксис, типы данных, функции
- SQL: SELECT, JOIN, индексы, оптимизация запросов
- Django: ORM, модели, миграции, views
- PostgreSQL: установка, базовая конфигурация, типы данных
- Docker: образы, контейнеры, docker-compose

**Маршрут:**
- Python синтаксис → пишешь скрипт с функциями, циклами, обработкой исключений, работаешь со списками и словарями
- SQL запросы → выбираешь данные из таблицы, объединяешь две таблицы через JOIN, фильтруешь по условиям, сортируешь результаты
- Django модели → создаёшь модель User с полями, запускаешь миграцию, добавляешь запись в БД через ORM, фильтруешь записи
- PostgreSQL → подключаешься к БД, создаёшь таблицы, вставляешь данные, видишь их в pgAdmin
- Docker → собираешь образ Python-приложения, запускаешь контейнер, приложение работает изолированно от системы

**Milestone:** Могу создать Django-приложение с моделью, сохранить данные в PostgreSQL и запустить всё в Docker-контейнере


**Практика:**
- **Калькулятор с историей** — Пишешь Python-скрипт с функциями сложения, вычитания, умножения. Сохраняешь каждую операцию в список, выводишь историю последних 10 операций
- **Управление списком задач через SQL** — Создаёшь таблицу tasks с полями id, title, status, created_at. Добавляешь задачи, обновляешь статус, удаляешь выполненные, выбираешь активные задачи отсортированные по дате
- **Django-приложение каталога книг** — Создаёшь модель Book с полями title, author, year, price. Добавляешь 5 книг через Django ORM, фильтруешь по году и автору, выводишь список в консоль

### Фаза 2: Practice (30 дней)

**Темы:**
- Django REST Framework: сериализаторы, viewsets, маршруты
- Аутентификация и авторизация: JWT, permissions
- Кэширование: Redis, cache-стратегии
- Тестирование: unittest, pytest, fixtures
- CI/CD: GitHub Actions, автоматизация тестов
- FastAPI: основы, async/await, Pydantic

**Маршрут:**
- Django REST Framework → создаёшь API endpoint /api/books, возвращаешь JSON со списком книг, фильтруешь по автору через query-параметры
- Сериализаторы → преобразуешь объект Book в JSON и обратно, валидируешь входные данные, обрабатываешь ошибки
- JWT авторизация → регистрируешь пользователя, получаешь токен, отправляешь токен в заголовке Authorization, сервер проверяет и разрешает доступ
- Redis кэширование → кэшируешь список книг на 5 минут, при повторном запросе берёшь из кэша вместо БД, видишь ускорение ответа
- Pytest → пишешь тесты для API: проверяешь статус-коды, структуру JSON, авторизацию, видишь отчёт о покрытии
- GitHub Actions → при push в main автоматически запускаются тесты, если тесты падают — merge блокируется

**Milestone:** Могу написать REST API с аутентификацией, кэшированием и покрытием тестами, которые автоматически запускаются при push


**Практика:**
- **REST API интернет-магазина** — Создаёшь endpoints для товаров: GET /api/products (с фильтром по цене и категории), POST /api/products (добавление), PUT /api/products/{id} (редактирование), DELETE /api/products/{id}. Добавляешь JWT авторизацию — только админ может добавлять товары
- **Кэширование популярных товаров** — Кэшируешь в Redis топ-10 товаров по продажам на 1 час. Первый запрос идёт в БД (медленно), второй берёт из кэша (быстро). Пишешь тесты, проверяющие что кэш работает
- **CI/CD pipeline для API** — Настраиваешь GitHub Actions: при push запускаются pytest, проверяется код линтером (flake8), если всё ОК — деплоится на staging. Видишь статус в pull request

### Фаза 3: Portfolio (30 дней)

**Темы:**
- Микросервисная архитектура: разделение на сервисы
- Kubernetes: деплой контейнеров, масштабирование
- Yandex Cloud / AWS: облачные БД, S3, API Gateway
- Асинхронные задачи: Celery, RabbitMQ
- Мониторинг и логирование: Prometheus, ELK Stack
- Оптимизация БД: индексы, query optimization, N+1 проблема

**Маршрут:**
- Асинхронные задачи → при заказе товара отправляешь письмо пользователю через Celery (не блокируя ответ), письмо приходит через 5 секунд
- Kubernetes → упаковываешь API в Docker, деплоишь в K8s, видишь 3 реплики приложения, при нагрузке автоматически создаётся 4-я реплика
- Yandex Cloud → загружаешь фотографии товаров в Object Storage (S3), генерируешь публичные URL, фотографии отображаются на фронте
- Оптимизация запросов → находишь N+1 проблему (запрос за товаром + запрос за каждой категорией), используешь select_related, видишь что запросов стало 2 вместо 50
- Мониторинг → настраиваешь Prometheus для сбора метрик (время ответа, количество ошибок), видишь графики в Grafana, получаешь алерт если API медленнее 500ms

**Milestone:** Могу спроектировать и развернуть высоконагруженное приложение с микросервисами, асинхронными задачами, автоматическим масштабированием и мониторингом


**Практика:**
- **Микросервисная архитектура e-commerce** — Разделяешь монолит на 3 сервиса: Products API, Orders API, Notifications Service. Каждый в отдельном Docker-контейнере. Orders Service вызывает Products Service через HTTP, при создании заказа отправляет задачу в Celery для отправки письма
- **Деплой в Kubernetes с автомасштабированием** — Пишешь Kubernetes manifests (deployment, service, hpa). Деплоишь API, настраиваешь HorizontalPodAutoscaler на 80% CPU. Генерируешь нагрузку через Apache Bench, видишь как создаются новые поды
- **Оптимизация и мониторинг production API** — Находишь N+1 проблемы в запросах, добавляешь индексы в БД, кэшируешь часто запрашиваемые данные. Настраиваешь Prometheus для сбора метрик, создаёшь Grafana dashboard с графиками времени ответа и количества ошибок

## Портфолио-проект

### FinTracker API — платформа для отслеживания расходов и бюджета

**Проблема:** Пользователи не видят куда уходят деньги, не могут спланировать бюджет и не понимают свои траты. Нужна система которая автоматически категоризирует расходы, показывает аналитику и предупреждает о превышении лимитов

**Сценарии использования:**
- Регистрируюсь через API, получаю JWT токен, использую его для всех запросов
- Добавляю расход (сумма, категория, дата, описание), система сохраняет в БД и возвращает JSON с ID расхода
- Запрашиваю GET /api/expenses?month=2026-01&category=food, получаю список расходов за январь по категории еда с суммой
- Запрашиваю GET /api/analytics/monthly, получаю график расходов по месяцам и категориям в JSON
- Устанавливаю лимит на категорию (food: 5000 руб/месяц), при добавлении расхода система проверяет лимит и отправляет уведомление если превышен

**Технические задачи:**
- Django ORM + PostgreSQL — оптимизация запросов для аналитики (агрегация по месяцам и категориям без N+1 проблемы, использование select_related и prefetch_related)
- Redis кэширование — кэшируешь аналитику пользователя на день, при добавлении нового расхода инвалидируешь кэш
- Celery + RabbitMQ — асинхронная отправка уведомлений по email/SMS при превышении лимита (не блокируя API ответ)
- JWT авторизация — каждый пользователь видит только свои расходы, используешь permission classes для проверки прав
- Kubernetes деплой — упаковываешь API, Celery worker и PostgreSQL в отдельные контейнеры, деплоишь в K8s с HPA для масштабирования при нагрузке

**Навыки в проекте:**
- Django REST Framework — создание REST API с viewsets, сериализаторами, фильтрацией и пагинацией
- PostgreSQL + Django ORM — проектирование схемы БД (User, Expense, Category, Budget), оптимизация запросов через select_related
- JWT авторизация — регистрация, логин, выдача токенов, проверка прав доступа через permission classes
- Redis — кэширование аналитики и лимитов пользователя для ускорения ответов
- Celery + RabbitMQ — асинхронные задачи для отправки уведомлений и расчёта аналитики в фоне
- Docker + docker-compose — упаковка API, Celery, Redis, PostgreSQL в контейнеры для локальной разработки
- Kubernetes — деплой приложения в K8s, настройка HPA для автомасштабирования, использование ConfigMap для конфигурации
- GitHub Actions — автоматический запуск тестов при push, проверка кода линтером, деплой на staging
- Pytest — написание unit-тестов для API endpoints, интеграционных тестов с БД, fixtures для подготовки тестовых данных
- Мониторинг — настройка Prometheus для сбора метрик (время ответа, количество ошибок), Grafana dashboard для визуализации
